Инсталляция Plazma ERP+CRM





Требования к аппаратному обеспечению

Клиент

Процессор: Типа Pentium 633 MHz или выше

Дисковое пространство: 50 MB

Оперативная память: 128 MB (256 MB рекомендуется)

Сетевой адаптер 10 Mb (100 Mb рекомендуется)



Сервер

Процессор: Типа Pentium 1700 MHz или выше

Дисковое пространство: 100 MB

Оперативная память: 256 MB (512 MB рекомендуется)

Сетевой адаптер 10 Mb (100 Mb рекомендуется)



Клиент+Сервер

Данный вариант используется для однопользовательского, малопользовательского (до 3 рабочих станций) или демо режима

Процессор: Типа Pentium 1700 MHz или выше

Дисковое пространство: 50 MB

Оперативная память: 128 MB (256 MB рекомендуется)

Сетевой адаптер 10 Mb (100 Mb рекомендуется)





Требования к программному обеспечению

Операционная система: Windows, Linux, MacOS

База данных: Oracle 10g, PostgreSQL 8.x, MySQL 5.x, Firebird 1.5.3 или выше, HSQL 1.7.2 или выше

Для запуска программы инсталляции необходимо наличие установленной JRE (Java Runtime Environment) версии 1.5 или выше на Вашем компьютере, если только сама инстялляция не включает JRE (это зависит от типа инсталляции) Скачать последнюю версию JRE можно здесь.


ВАЖНО! Только база данных HSQL входит в комплект инсталляции Plazma ERP+CRM. Если Вы будете использовать другую базу данных, то Вам необходимо самостоятельно загрузить и установать на Вашем компьютере сервер баз данных.



Типы инсталляций

№НазваниеОбозначение в имени файла инсталляцииОписание
1Standalone (Client+Server)Общая часть
2ClientclientКлиентская часть
3ServerserverСерверная часть

Вариант 1 (Standalone) работает БЕЗ СЕРВЕРА ПРИЛОЖЕНИЙ, тоесть клиент напрямую взаимодействует с базой данных. Этот вариант рекомендуеться как начальный для: однопользовательского, малопользовательского (до 3 рабочих станций) или демо режима.

Имя файла инсталляции Plazma приложения представляется в виде:

plazma-<тип инсталляции>-<версия>-<операционная система>-installer.<расширение файла>

где .<расширение файла> может принимать значения .jar или .exe



Загрузка

Инсталляцию Plazma ERP+CRM можно загрузить из сайта http://plazma.sourceforge.net

В зависимости от операционной системы и серверной технологии, которая будет использоваться, Вы можете выбрать различные варианты инсталляции:

Windows (Standalone)

№Файл
1plazma-<версия>-win32-installer.jar
2plazma-<версия>-win32-installer.exe
3plazma-<версия>-win32-jre-installer.exe


Windows (Lite Server)

№Файл
1plazma-client-lite-<версия>-win32-installer.exe
2plazma-server-lite-<версия>-win32-installer.jar




Linux (Standalone)

№Файл
1plazma-<версия>-linux-gtk-installer.jar


Linux (Lite Server)

№Файл
1plazma-client-lite-<версия>-linux-gtk-installer.jar
2plazma-server-lite-<версия>-linux-gtk-installer.jar


где <версия> - версия программы, например, 0.1.8



Инсталляция

Windows

В зависимости от расширения файла инсталляции, необходимо выполнить следующие действия:

.exe - Запустить файл на выполниние
.jar - В командной строке набрать: java -jar <имя jar файла>

Linux

В командной строке набрать: java -jar <имя jar файла>



Если выбран вариант инсталляции с JRE, то вначале будет запущена становка JRE, а потом уже сама установка Plazma ERP+CRM
В процессе установки Plazma ERP+CRM необходимо вначале выбрать язык и каталог установки, далее необходимые компоненты установки.

Если выбран вариант инсталляции с использованием сервера приложений (Lite Server), то вначале необходимо выполнить инсталляцию сервера, а потом - клиента





Запуск

Для запуска Standalone или Client приложения необходимо произвести следующие действия:


Windows

Меню Пуск -> Программы -> Plazma ERP+CRM <версия> -> Plazma ERP+CRM


или находясь в корневой папке программы запустить на выполнение файл

plazma.bat


или выполнить команду

javaw -Djava.library.path="lib\swt\win32" -jar launcher.jar plazma


Linux

находясь в корневой папке программы запустить на выполнение файл

plazma.sh


или выполнить команду

java -Djava.library.path="lib/swt/linux-gtk" -jar launcher.jar plazma


ВАЖНО! Если выбран вариант инсталляции с использованием сервера приложений (Lite Server), то вначале необходимо запустить сервер, а потом клиент.
Подробные инструкции по запуску программы смотрите в разделе Начало работы



Для старта Lite Server необходимо выполнить командный файл:

Windows

<PlazmaLiteServer>/start_server.bat

Linux

<PlazmaLiteServer>/start_server.sh
