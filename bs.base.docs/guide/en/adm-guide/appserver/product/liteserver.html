Lite Server



Common

Plazma Lite Server is a lite server application using RMI and Spring framework.

Server


By default the server uses 2005 and 2007 ports.
2005 is the service port (call services)
2007 is the command port (call commands)
If your system already uses data ports, it is necessary to change the server configuration.
You can set the service port in file <PlazmaServer>/conf/application-context-rmi.xml: 
<bean class="org.springframework.remoting.rmi.RmiServiceExporter">
    <property name="serviceName"><value>ServiceInvoker</value></property>
    <property name="service"> org.plazmaforge.framework.platform.service.ServiceInvoker</value></property>
    <property name="registryPort"><value>2005</value></property>
</bean>
IMPORTANT! If you change the service port on the server then you have to change the service port on the client!
The ports on the server and the client must be equal.

On client you can set the service port in file <PlazmaClient>/conf/rmi.properties
See configuring client.

To start the Lite server you have to run the follow command file: 


   Windows
   -------

   start_server.bat


   Linux
   -----

   start_server.sh

You can send the command port as a parameter when you start the server: 

   start_server -commandport 2007
Connecting to the database is configured in <PlazmaServer>/conf/jdbc.properties
See Database

Client

You have to configure RMI in file <PlazmaClient>/conf/rmi.properties 


rmi.host = localhost
rmi.port = 2005
rmi.host - your server name (by defalut localhost), rmi.port - your service prot (by default 2005)

IMPORTANT! The ports on the server and the client must be equal.
